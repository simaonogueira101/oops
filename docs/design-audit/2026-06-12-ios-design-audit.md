# Oops iOS Design Audit — Synthesis Report

*22 audit agents: 4 design skills (hig, ios-liquid-glass, ios-design-consultant, ios-ui-craft) × 5 areas, plus 2 supplemental skill passes (swiftui-ui-patterns, swiftui-liquid-glass). Findings merged by underlying issue; agreement counts = distinct skills that independently flagged it.*

## Executive summary

The app's foundations are sound — one shared `Card`, a disciplined 6-hue token palette, deterministic mock data — but the "not very Apple-like" feel traces to a small number of *systemic* choices in shared files, which is good news: most fixes are one-file edits that every screen inherits. The strongest cross-skill signals are: (1) the drawer presentation layer strips the system sheet experience (`presentationBackground` override, no titles, no close buttons, no in-drawer push registration — flagged in some form by 5 of 6 skills); (2) the custom `TopBar` forfeits system navigation bars, large titles, and the iOS 26 scroll-edge treatment; (3) the `Card` chrome itself (ALL-CAPS tracked micro-labels, 0.5pt stroke outlines, 18pt radius, domain-tinted numbers) is the Whoop/Oura idiom rather than Apple Health's; (4) fixed-point hero fonts and frozen chart frames defeat Dynamic Type; and (5) several controls are visibly dead (the Day/Week/Month/Year picker changes nothing on any screen). Per project constraints, all recommendations to convert the drawer model to `NavigationStack` pushes have been **dropped** (drawer navigation is a deliberate choice — the ~10 findings urging push conversion were instead redirected into their drawer-compatible sub-fixes: native sheet material, task titles, close buttons, and pushing *within* the drawer's own stack). All color fixes below stay inside the reduced 6-hue palette (they relocate or ramp existing hues; none add new ones), and nothing touches the local-only data constraint.

## Top 10 changes

1. **Make drawers behave like real system sheets** *(5 skills, 10+ findings — strongest signal in the audit)*
   - **Issue:** Every drawer is stripped of the iOS 26 Liquid Glass sheet treatment by `.presentationBackground(AppColor.surfaceElevated)`; sheets have no task title and no dismiss button (grabber only); the drag indicator is forced visible on non-resizable `[.large]`-only sheets; and the drawer's embedded `NavigationStack` has **zero `.navigationDestination` registrations** (grep-verified), so tapping a card inside a drawer stacks a second full-height sheet on top of the first.
   - **Fix:** In `cardDrawer`: delete the `.presentationBackground(AppColor.surfaceElevated)` line (if contrast is needed, use `.scrollContentBackground(.hidden)` + `.background(AppColor.background)` on the *content*); change to `.presentationDragIndicator(detents.count > 1 ? .visible : .automatic)`; default detents to `[.medium, .large]` for glanceable drill-ins. In `DrawerLink`'s `NavigationStack`, add `.navigationDestination(for: AppRoute.self) { RouteDestination(route: $0) }` and make inner `navigates(to:)` emit `NavigationLink(value:)` when already inside a drawer (an `@Environment(\.isInsideDrawer)` flag) so depth-2 navigation pushes with a back button instead of sheet-on-sheet. Give every drawer root `.navigationTitle(...)` + `.toolbar { ToolbarItem(placement: .topBarTrailing) { Button(role: .close) { dismiss() } } }` (iOS 26 renders the standard circular glass ✕); Record form gets `.cancellationAction` + `.confirmationAction`.
   - **Files:** `Shared/DesignSystem/Card/CardModifiers.swift`, `Shared/Screens/AppRoute.swift`, `Shared/Screens/Workout/RecordWorkoutViews.swift`, `iOS/Home/ProfileView.swift`, `Shared/Screens/MetricDetailScreen.swift` (titles)
   - **Flagged by:** hig (all 4 areas), ios-liquid-glass (3 areas), ios-design-consultant, ios-ui-craft, swiftui-liquid-glass

2. **Adopt system navigation bars; fold TopBar into toolbars** *(4 skills)*
   - **Issue:** `TopBar` sits in a `VStack` above the `TabView`, so no screen has a navigation bar, large title, or scroll-edge effect — the single most recognizable stock-Apple cue is absent app-wide, and the chrome typography runs one text style too small (`.subheadline` date label vs the system's `.headline` title metric; `.imageScale(.small)` glyphs vs body-size bar buttons).
   - **Fix:** Wrap each `Tab`'s screen in `NavigationStack { … }` with `.navigationTitle("Summary"/"Sleep"/"Recovery"/"Strain")` (default large-title, auto-collapsing with the system glass edge effect). Move the avatar to `ToolbarItem(placement: .topBarTrailing)` at ~36pt (Health's placement — currently it's top-*leading* at 26pt), battery/sync into a trailing `ToolbarItemGroup`, and the date into `ToolbarItem(placement: .principal)` or `.navigationSubtitle(date.formatted(.dateTime.weekday(.wide).month().day()))`. Scope the date control to Overview only — it currently changes nothing on the other three tabs. Drawers remain the drill-in model; this fixes the chrome, not the navigation pattern.
   - **Files:** `iOS/Home/HomeRootView.swift`, `iOS/Home/TopBar.swift`, all `Shared/Screens/*` roots (titles)
   - **Flagged by:** hig (navigation, sleep, domains), ios-design-consultant, swiftui-liquid-glass, ios-liquid-glass

3. **De-theme the Card chrome: sentence-case headers, no outline, system surfaces** *(5 skills)*
   - **Issue:** Every card renders `Text(label.uppercased()).font(.caption2.weight(.semibold)).tracking(0.8)` — letter-spaced all-caps is the signature Whoop/Oura overline, repeated in `StatTile` and `CompositeHeroRing`. Every card also draws a `0.5pt strokeBorder(AppColor.separator)` at an 18pt radius; Health cards are borderless, separated purely by grouped-background contrast. The custom neutral hexes (opaque separator, hand-picked surface/background) lose the system's dark-mode elevation and Increase Contrast adaptivity.
   - **Fix:** In `Card.header`, drop `.uppercased()` and `.tracking(0.8)`; render `Label(label, systemImage: icon).font(.subheadline.weight(.semibold)).foregroundStyle(accent ?? AppColor.secondaryLabel)` (Health's icon + sentence-case header pattern); same in `StatTile`/`CompositeHeroRing`. Delete the `strokeBorder` overlay and the `Divider().overlay(AppColor.separator)`; change radius to `.rect(cornerRadius: 12, style: .continuous)` or `ConcentricRectangle()` for iOS 26 concentricity. Keep the token *names* but re-back the five neutrals with dynamic system colors: `background = Color(.systemGroupedBackground)`, `surface = Color(.secondarySystemGroupedBackground)`, `surfaceElevated = Color(.tertiarySystemGroupedBackground)`, `separator = Color(.separator)`, `track = Color(.systemFill)` (with `#if canImport(UIKit)` / NSColor equivalents). The 6-hue domain/status palette is untouched.
   - **Files:** `Shared/DesignSystem/Card/Card.swift`, `Shared/DesignSystem/Blocks/StatTile.swift`, `Shared/DesignSystem/Blocks/CompositeHeroRing.swift`, `Shared/DesignSystem/AppColor.swift` + `Shared/Assets.xcassets/Colors`
   - **Flagged by:** hig (home, domains, designsystem), ios-liquid-glass (home, designsystem), ios-design-consultant (sleep, domains)

4. **Restore Dynamic Type: scale the hero fonts and chart containers** *(5 skills)*
   - **Issue:** All four `Typography.swift` tokens are fixed (`Font.system(size: 56/84/34/28)`) so hero numbers freeze while body text scales — inverting hierarchy at accessibility sizes. Chart/ring containers are equally frozen (`SleepStageChart` 160pt, `LineTrendChart` 180, `BarSeriesChart` 160, `Sparkline` 40, `ScoreRing` 120/150, `GoalProgress` 96), clipping axis labels at large text sizes. `headerGlyph` (34) and `sectionGlyph` (28) literally re-implement `.largeTitle`/`.title` as non-scaling sizes.
   - **Fix:** Replace `headerGlyph`→`Font.largeTitle`, `sectionGlyph`→`Font.title`. For `metricValue`/`heroGlyph`, keep the token names but back them with `@ScaledMetric(relativeTo: .largeTitle) var size = 56` via a small `MetricValueText`/`.metricValueStyle()` helper in `Typography.swift` (single lint-exempt source of truth), and add `.monospacedDigit()`. Give each chart `@ScaledMetric(relativeTo: .body) private var chartHeight: CGFloat = 160` → `.frame(height: chartHeight)`; same for ring diameters (`relativeTo: .largeTitle`).
   - **Files:** `Shared/DesignSystem/Typography.swift`, `Shared/DesignSystem/Charts/SleepStageChart.swift`, `LineTrendChart.swift`, `BarSeriesChart.swift`, `Shared/DesignSystem/Blocks/Sparkline.swift`, `ScoreRing.swift`, `GoalProgress.swift`
   - **Flagged by:** hig (all 4 areas), ios-design-consultant, ios-liquid-glass (sleep)

5. **Color discipline: numbers in label color, one app-wide control tint** *(5 skills)*
   - **Issue:** Domain hues are painted onto data values themselves (`compactValue`, `StatTile` values, `HeroNumber`, `CompositeHeroRing` satellites — three competing hues inside one card), CTA buttons are tinted per-domain (a Sleep card gets an indigo button, a Strain card an orange one), and tinting is positional (only the *first* tile in every stat row is tinted). There is also no `AccentColor` asset, so the macOS target and any unplumbed control falls back to system blue next to the Recovery hue.
   - **Fix:** Health's distribution: values in `AppColor.label`/`.primary`; the domain hue lives only in the header label/icon, ring, and chart marks. Change `compactValue` to `.foregroundStyle(AppColor.label)` and pass `color` only to the `Sparkline`; remove `accent` from `StatTile` values and the position-one tints in `StrainView`/`WorkoutDetailView`/`MetricDetailScreen`; in `Card`'s footer CTA use `.tint(AppColor.accent)` unconditionally. Add `AccentColor.colorset` (Recovery values) + `ASSETCATALOG_COMPILER_GLOBAL_ACCENT_COLOR_NAME: AccentColor` on both targets in `project.yml`, then `xcodegen generate`; make `AppColor.accent = Color.accentColor`. Zero palette changes — only which *layer* each hue may appear in.
   - **Files:** `Shared/Screens/OverviewView.swift`, `Shared/DesignSystem/Blocks/StatTile.swift`, `HeroNumber.swift`, `CompositeHeroRing.swift`, `Shared/DesignSystem/Card/Card.swift`, `Shared/DesignSystem/AppColor.swift`, `project.yml`
   - **Flagged by:** hig (home), ios-ui-craft (designsystem), ios-design-consultant (designsystem), ios-liquid-glass (domains), swiftui-ui-patterns

6. **Replace the self-described "Whoop-style" hero with the Fitness Summary idiom** *(2 skills — but it violates the "never Whoop styling" hard constraint)*
   - **Issue:** `CompositeHeroRing`'s own doc comment says "(Whoop-style overview)": a center ring flanked by four differently-tinted satellite stat columns. It is the first card on screen and sets the whole app's tone; no Apple app composes a hero this way.
   - **Fix:** `HStack(spacing: Spacing.lg) { ScoreRing(score:accent:size: 120); VStack(alignment: .leading, spacing: Spacing.sm) { ForEach(stats) { LabeledContent(stat.label) { Text(stat.value).font(.headline).monospacedDigit() } } } }` — label `.subheadline`/`.secondary`, value `.primary`; if a stat needs domain identity, a small tinted SF Symbol (e.g. `bed.double.fill` in `AppColor.sleep`), not a tinted number. The `leading`/`trailing` arrays collapse into one `stats` array.
   - **Files:** `Shared/DesignSystem/Blocks/CompositeHeroRing.swift`, `Shared/Screens/OverviewView.swift`
   - **Flagged by:** ios-liquid-glass (home, designsystem)

7. **Wire the Day/Week/Month/Year picker to the data — it is a dead control on every screen, and pin it to the control layer** *(4 skills)*
   - **Issue:** Every chart renders a fixed 14-day series regardless of selection (`MetricDetailScreen`, `HRZonesView`, Sleep/Recovery/Strain trends cards) — a visibly non-functional native control is one of the fastest un-Apple tells. It is also embedded *inside* a content card and scrolls away, where Health pins it beneath the nav bar.
   - **Fix:** Add `var days: Int { switch self { case .day: 1; case .week: 7; case .month: 30; case .year: 365 } }` to `Period`; recompute series from `period` (e.g. `mock.series(days: period.days, …)`), adjust `.chartXAxis { AxisMarks(values: .stride(by: period == .year ? .month : .day)) }`, animate with `.animation(.snappy, value: period)` + `.sensoryFeedback(.selection, trigger: period)`. Hoist the picker out of the Card: `ScrollView { … }.safeAreaInset(edge: .top) { PeriodPicker(period: $period) }` + `.scrollEdgeEffectStyle(.soft, for: .top)`.
   - **Files:** `Shared/DesignSystem/Blocks/PeriodPicker.swift`, `Shared/Screens/MetricDetailScreen.swift`, `Shared/Screens/Strain/HRZonesView.swift`, `Shared/Screens/Sleep/SleepView.swift`, `Recovery/RecoveryView.swift`, `Strain/StrainView.swift`
   - **Flagged by:** ios-liquid-glass (sleep, domains), ios-design-consultant (domains, sleep), ios-ui-craft (sleep), hig

8. **Fix cross-domain hue bleed with single-hue intensity ramps** *(4 skills)*
   - **Issue:** `SleepStage.color` maps Awake→`AppColor.strain` and Light→`AppColor.recovery`, so on the Sleep tab the recovery hue simultaneously means "light sleep", "heart rate" (chart two cards below), and "tappable" (app tint). `hrZones()` colors intensity with the *status* trio (Peak = the app-wide error red) plus the Recovery hue inside the Strain tab. Same hue, three meanings — color stops carrying information.
   - **Fix:** Stay inside the palette with opacity/lightness ramps: sleep stages as a ramp of the sleep hue (`.deep` = `AppColor.sleep`, `.light` = `.sleep.opacity(0.7)`, `.rem` = `.sleep.opacity(0.45)`) with `AppColor.caution` for `.awake` (the documented attention color); HR zones as a strain ramp (`AppColor.strain.opacity(0.35/0.55/0.75/1.0)` for Light→Peak). Status trio goes back to meaning only good/warn/bad app-wide. Zero new hues.
   - **Files:** `Shared/Model/HealthModels.swift`, `Shared/Model/MockHealthData.swift`
   - **Flagged by:** hig (sleep), ios-ui-craft (designsystem), ios-design-consultant (domains), ios-liquid-glass (domains)

9. **Route all numbers and durations through Foundation format styles** *(4 skills)*
   - **Issue:** "12000 steps", "9240 / 12000", bare "430" calories, and a hand-rolled `hm()` helper duplicated in `SleepView`, `StrainView`, and `WorkoutsView` producing "7h 32m" — ungrouped, unlocalized digits are an instantly recognizable non-Apple tell (Health shows "7 hr 32 min", "12,000").
   - **Fix:** `Text(metrics.steps, format: .number)`; `Duration.seconds(ti).formatted(.units(allowed: [.hours, .minutes], width: .abbreviated))` (or `.narrow` where compact) replacing both `hm()` copies; `Measurement(value:, unit: UnitEnergy.kilocalories).formatted(.measurement(width: .abbreviated, usage: .workout))` for calories; `Text(fraction, format: .percent.precision(.fractionLength(0)))` in `GoalProgress`; `Text(timerInterval: active.startDate...Date.distantFuture)` for the live elapsed timer (also deletes the per-second `TimelineView` re-render). Change `StatTile`/`HeroNumber` to accept `Text` or value+`FormatStyle` so formatting can't be skipped at call sites.
   - **Files:** `Shared/DesignSystem/Blocks/GoalProgress.swift`, `StatTile.swift`, `ZoneScale.swift`, `Shared/Screens/Strain/StrainView.swift`, `WorkoutsView.swift`, `Sleep/SleepView.swift`, `Workout/RecordWorkoutViews.swift`, `Settings/SettingsView.swift`
   - **Flagged by:** swiftui-ui-patterns, ios-ui-craft (domains, home), ios-design-consultant (home), hig (sleep)

10. **Accessibility pass: labels, combined elements, real buttons, 44pt targets** *(5 skills)*
    - **Issue:** Every icon-only chrome control is unlabeled to VoiceOver (avatar, sync, date chevrons, banner dismiss); `ScoreRing` reads as raw SectorMark values, `DeltaLabel` reads "arrow up, 44" (the *baseline*, not the change), breakdown rows are four fragments; `ExpandableCard` toggles via `onTapGesture` with no button trait, state announcement, or haptic; the toggle accessory is `Toggle("", …)` (anonymous switch); chevrons/avatar/banner-dismiss are ~17–26pt targets vs the 44×44pt minimum.
    - **Fix:** `.accessibilityLabel` on every icon-only control (DateSelector collapsed to one element with `.accessibilityAdjustableAction`); `.accessibilityElement(children: .ignore)` + label/value on `ScoreRing` ("Score, 86 out of 100"), batteryLabel, and `.accessibilityElement(children: .combine)` on stat/breakdown rows; make `ExpandableCard` a real `Button` with `.accessibilityValue(expanded ? "Expanded" : "Collapsed")`, a single rotating chevron (`.rotationEffect(.degrees(expanded ? 180 : 0))`), and `.sensoryFeedback(.selection, trigger: expanded)`; `case toggle(String, Binding<Bool>)` + `Toggle(label, isOn:).labelsHidden()`; `.frame(minWidth: 44, minHeight: 44).contentShape(Rectangle())` on all small buttons. `DeltaLabel`: render the signed delta (`format: .number.sign(strategy: .always())`) so text, not hue, carries the meaning.
    - **Files:** `iOS/Home/TopBar.swift`, `iOS/Home/UpdatedBanner.swift`, `Shared/DesignSystem/Card/CardModifiers.swift`, `Card.swift`, `Blocks/ScoreRing.swift`, `DeltaLabel.swift`, `DateScroller.swift`, `Charts/SleepStageChart.swift`
    - **Flagged by:** ios-ui-craft (navigation), ios-design-consultant (designsystem, navigation), ios-liquid-glass (sleep, designsystem), hig (navigation)

## Remaining findings by area

**Navigation / chrome**
- Record "+" is an action masquerading as a tab in the `role: .search` slot, with a ghost-tab mirror that double-instantiates the active screen for a frame; replace with a toolbar `+` or `.tabViewBottomAccessory`, deleting the `tabSelection` binding hack *(hig, swiftui-liquid-glass)*.
- Domain-tab routes (`.sleep`/`.recovery`/`.strain`) present a live *duplicate* of an existing tab in a sheet; resolve those routes to tab switches and reserve drawers for routes without a tab *(swiftui-liquid-glass)*.
- Presentation state is fragmented — every card owns a private `@State` + `.sheet`; hoist to one `@State var route: AppRoute?` + single `.sheet(item:)` per screen, which also enables `.matchedTransitionSource` / `.navigationTransition(.zoom)` morphs for Profile/Sync sheets *(ios-ui-craft, ios-liquid-glass)*.
- Date control: stepper-only (14 taps to go back two weeks) — make the label open a `.graphical` `DatePicker`; UpdatedBanner reflows the whole UI — present as `.overlay(alignment: .top)`; battery cluster is inert next to an identically-styled tappable sync button — surface the existing `BatteryScreen` *(ios-design-consultant, ios-liquid-glass, ios-ui-craft)*.

**Home / Overview**
- Hidden day-swipe gesture rebuilds the whole feed via `.id(dayKey)` identity reset; use a paging `ScrollView` (`.scrollTargetBehavior(.paging)` + `.scrollPosition(id:)`) or in-place animation with `.contentTransition(.numericText())` *(ios-design-consultant)*.
- Cards chart the wrong data: Sleep card plots `hrvSeries`, Strain card plots `stepsSeries`, Heart Rate hardcodes "now 61 bpm" — add real per-metric series to `MockHealthData` *(ios-design-consultant)*.
- Two-up Sleep/Strain row has ragged bottoms; prefer Health's single column of full-width cards *(ios-design-consultant)*.
- Active-workout banner scrolls away inside the feed; move to `.tabViewBottomAccessory` (Now Playing-style persistent bar) *(ios-liquid-glass)*.
- `GoalProgress` ring → native `Gauge(...).gaugeStyle(.accessoryCircularCapacity)` *(swiftui-ui-patterns)*.

**Sleep**
- Hypnogram: pin `.chartXScale(domain: start...end)`, restore `AxisGridLine()`/`AxisTick()` in the custom axis, and add `RuleMark` stage-transition connectors for Health's staircase look *(ios-ui-craft, ios-design-consultant)*.
- Add `.chartXSelection` scrubbing with a value callout to the hypnogram and trend charts — Health's defining chart interaction *(ios-liquid-glass, ios-design-consultant)*.
- Hardcoded headline literals disagree with charted data (score 86, "52 bpm" accessory vs jittered series); derive from one stored series; legend should come from `.chartForegroundStyleScale` instead of hand-drawn swatches and duplicated `order` arrays *(ios-ui-craft, ios-liquid-glass)*.
- No empty state: guard with `ContentUnavailableView("No Sleep Data", systemImage: "moon.zzz", …)`; hoist `MockHealthData` to `@State` and give models date-based identity so period changes can morph *(ios-ui-craft)*.

**Recovery / Strain (domains)**
- Card↔destination mismatches: "Resting heart rate 54" opens "Heart Rate 61"; "Body temperature" opens "Skin Temperature" — one metric descriptor as source of truth *(ios-design-consultant)*.
- `ZoneScale`'s gradient bar encodes no data and `HRZonesView` merely repeats the card that opened it — proportional time-in-zone segments (Chart `BarMark` widths by minutes) *(3 skills)*.
- `WorkoutsView` reinvents list rows as bordered mini-cards — use `List`/`.insetGrouped`; its detail HR chart plots a 20-*day* series for a minutes-long workout — generate samples across the workout window with `.chartXScale(domain:)` *(ios-liquid-glass, ios-ui-craft)*.
- "End Workout" is destructive with no confirmation, mid-content at the medium detent — gate with `.confirmationDialog` and pin via `.safeAreaInset(edge: .bottom)` *(ios-design-consultant)*.
- Record form: hand-rolled checkmark rows → native inline `Picker`; mixed accents (strain rows, recovery checkmark) → one `.tint(AppColor.strain)` on the Form; Start button's clear-list-row hack → `.safeAreaBar(edge: .bottom)` + `.glassProminent` *(ios-liquid-glass ×2, hig)*.
- `ProfileView` mixes live (photo) and deferred (name) commit, and swipe-dismiss silently discards typed edits — pick one model *(ios-ui-craft)*.

**Design system**
- `RingChart`'s two-SectorMark ring has visible seams; replace with `Circle().trim(to:)` + round caps and a point-value `lineWidth` *(ios-liquid-glass)*.
- `CardStyle.tinted` diagonal gradient wash → flat `color.opacity(0.12)` or tint only the label/CTA *(ios-liquid-glass)*.
- Mid-scale value typography is improvised five ways on one screen; add `cardValue`/`cardValueSecondary` tokens to `Typography.swift` *(ios-liquid-glass)*.
- Asset palette lacks high-contrast variants, and light-mode Caution (~3.2:1) fails 4.5:1 as caption text — add contrast appearances and darken toward ~#A66A00 *(ios-ui-craft)*.
- `StatTile` should take an optional small-baseline `unit:` parameter ("118" + "BPM") instead of unit text baked into the value string *(ios-ui-craft)*.

No finding proposed breaking the 6-hue palette or local-only data; color fixes above are hue-relocations and opacity ramps within the existing tokens.

## Appendix — patrickserrano skill test verdicts

### ps/swiftui-ui-patterns

The skill is accurate but architectural, not visual: it covers MV-over-MVVM, sheet(item:) patterns, Observation-era state wrappers, and TabView/NavigationStack scaffolding — all reasonable and current through iOS 17-era APIs, but it says nothing about HIG sizing, typography, color semantics, materials, charts, or anything iOS 26/Liquid Glass, which is precisely what this 'make it look stock Apple' audit needed. Of my five findings, only the native-first stance and the accessibility bullet gave even indirect support; the substantive rules (trailing chart axes, .controlSize(.large), Gauge styles, Text(_, format:), chevron.forward) all came from HIG/SDK knowledge outside the skill. Verdict: not outdated, but generic — as a design-audit rubric it's the wrong tool, and it should either gain a 'native look and feel' section (system controls, sizes, formatting, RTL, Dynamic Type) or be paired with a separate HIG-focused skill.

### ps/swiftui-liquid-glass

The skill is accurate and current as an API cheat-sheet — glassEffect, GlassEffectContainer, glassEffectID, .buttonStyle(.glass/.glassProminent), shape options and modifier ordering are all real iOS 26 APIs described correctly. But as an audit rubric it's shallow: it never says *where* Liquid Glass belongs (the navigation/chrome layer, not content) nor that system components — toolbars, tab bars, sheets — provide glass for free, which is the actual fix for this app; applied literally it would encourage glassing the opaque Health-style content cards, the opposite of Apple's guidance. It also omits the iOS 26 chrome APIs that matter most here (tabViewBottomAccessory, tabBarMinimizeBehavior, scrollEdgeEffectStyle, matchedTransitionSource), so 4 of my 5 findings came from HIG knowledge rather than the skill — only the glassProminent/morphing fix language drew directly from it.
